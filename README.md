# PlantCare  
**Система автоматизации ухода за растениями в ботаническом саду**


Это клиент-серверное веб-приложение, предназначенное для учета растений, планирования и контроля операций ухода, а также фиксации наблюдений за состоянием растений.


Система позволяет сотрудникам ботанического сада просматривать карточки растений, управлять задачами ухода, фиксировать проблемы и вести историю наблюдений.

---

# Архитектура системы

Система состоит из трех основных компонентов:

- Клиентское приложение — веб-интерфейс для пользователей системы  

- Серверное приложение — REST API и бизнес-логика  

- База данных — хранение информации о пользователях, растениях и задачах  


# Серверная часть 
Реализована с использованием FastAPI и предоставляет REST API для взаимодействия клиентского приложения с базой данных.

Используемые технологии: Python, FastAPI, SQLAlchemy, PostgreSQL, Uvicorn. 


# Описание основных папок и файлов

config.py - конфигурация приложения

database.py - настройка подключения к бд

dependencies.py - зависимости FastAPI

main.py - основной файл запуска FastAPI приложения

models/ - содержит модели бд SQLAlchemy

schemas/ - содержит схемы данных Pydantic

routers/ - содержит маршруты API

services/ - содержит бизнес-логику приложения

utils/ - содержит вспомогательные функции

# Клиентская часть (Frontend)

Реализована как веб-приложение, предоставляющее пользовательский интерфейс для работы с системой.

Используемые технологии: JavaScript, React, Vite, Axios.

# Описание основных папок и файлов

src/` — основная папка исходного кода клиентского приложения

main.jsx — точка входа приложения

App.jsx — основной компонент приложения

api/ — функции для взаимодействия с серверным API

components/ — переиспользуемые компоненты интерфейса

pages/ — страницы приложения

utils/ — вспомогательные функции

public/ — статические файлы (изображения, иконки)

vite.config.js — конфигурация сборщика Vite


# Запуск системы через Docker Compose
Для запуска проекта требуется:

- Docker

- Docker Compose

## Шаг 1. Клонировать репозиторий

git clone <URL_репозитория>

cd <папка_проекта>

## Шаг 2. Запустить систему

docker compose up --build

После запуска будут автоматически подняты три сервиса:

db — база данных PostgreSQL

server — сервер FastAPI

client — клиентское приложение React

## Доступ к системе

Клиентское приложение: ttp://localhost:5173

Сервер API: http://localhost:8000

Swagger UI: http://localhost:8000/docs

ReDoc; http://localhost:8000/redoc


## Остановка системы

Остановка контейнеров:

docker compose down

## Полная очистка данных базы

Удалиение контейнеров и данных базы:

docker compose down -v
